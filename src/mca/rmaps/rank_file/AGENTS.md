# AGENTS.md — `rmaps/rank_file` (explicit rankfile mapper)

Component guide for `src/mca/rmaps/rank_file/`. Read the
[framework guide](../AGENTS.md) first.

---

## Role and priority

rank_file implements **`--map-by rankfile:FILE=<path>`**: the user hands
in a file that names, for **every rank**, the host and the exact cpuset
(slot list) to bind it to. Lines look like:

```
rank 0=host1 slot=0-3
rank 1=host1 slot=4-7
rank 2=+n1   slot=1:0-1     # +nK = relative node index into the allocation
```

It is priority **95** — the second-highest rmaps mapper (only lsf, which
supplies its own affinity file, outranks it). Claimed only when the
mapping policy is `PRTE_MAPPING_BYUSER` (which `--map-by rankfile` sets)
and a rankfile path is available.

Files:

| File | Contents |
|------|----------|
| `rmaps_rank_file_component.c` | Registration, `priority` param (default 95), `query`; also the `slot_list` MCA param. |
| `rmaps_rank_file.c` | `prte_rmaps_rf_map()` and the rankfile parser `prte_rmaps_rank_file_parse()`. |
| `rmaps_rank_file_lex.l` / `rmaps_rank_file_lex.c` | Flex lexer for the rankfile grammar (`.c` is generated — edit the `.l`, not the `.c`). |
| `rmaps_rank_file.h` | `prte_rmaps_rank_file_map_t` (`{rank, node_name, slot_list[64]}`), `RMAPS_RANK_FILE_MAX_SLOTS`, lexer externs. |

## Two-phase structure

### Phase 1 — parse (`prte_rmaps_rank_file_parse`)
Drives the flex lexer over the rankfile and fills a `pmix_pointer_array_t
rankmap` indexed **by rank** with `prte_rmaps_rank_file_map_t` entries
(node name + slot-list string). It rejects unsupported tokens (quoted
strings, usernames), enforces `rank N=...` before `slot=...`, resolves
`localhost` to the HNP node name, strips FQDNs when
`!prte_keep_fqdn_hostnames`, and catches duplicate rank assignments via a
scratch `assigned_ranks_array`. `num_ranks` counts entries. **The lexer
`.c`/`.h` are generated — never hand-edit them; change `rmaps_rank_file_lex.l`
and regenerate.**

### Phase 2 — map (`prte_rmaps_rf_map`)
Gate: defer (`TAKE_NEXT_OPTION`) on restart, mismatched `req_mapper`,
non-`BYUSER` policy, or `options->ordered` (the ordered directive is
incompatible with an explicit rankfile). If no rankfile path is found
(per-app `PRTE_APP_MAP_FILE`, else job `PRTE_JOB_FILE`), returns
`PRTE_ERR_BAD_PARAM`. On acceptance it stamps `last_mapper` and sets
`options->map = PRTE_MAPPING_BYUSER`.

Then, per app (honoring `options->app_idx`), for each rank `k`:

1. Look up `rankmap[vpid_start + k]`.
   - **Present:** find its `node_name` in the node list (or resolve a
     `+nK` relative index into the list), take its `slot_list`.
   - **Absent:** the rank wasn't listed. Fall back to `options->cpuset`
     or `prte_hwloc_default_cpu_list` for the slots, and grab the next
     non-oversubscribed node (or least-loaded). A rank with neither a
     rankfile entry nor a default slot list is a `missing-rank` error.
2. `check_support`, `get_cpuset`, `check_avail`, `check_oversubscribed`,
   `setup_proc`.
3. **Assign the rank explicitly** (`proc->name.rank = rank`) and insert
   into `jdata->procs` — rank_file is a `userranked` mapper.
4. **Apply the slot list as the binding.** Parse `slots` with
   `prte_hwloc_base_cpu_list_parse()` into a bitmap, set
   `proc->cpuset`, verify the CPUs are actually available on the node
   (unless `overload`), and consume them from `node->available`. The
   proc's locale (`proc->obj`) is intentionally left NULL because a slot
   list may span multiple objects.

When not in per-app dispatch, `compute_vpids` runs to back-fill local/app
ranks (the vpids are already set).

## LSF affinity handling lives elsewhere

rank_file does **not** handle LSF affinity files. The live LSF-affinity
mapper is the standalone [`lsf`](../lsf/AGENTS.md) component (priority
100), which parses `LSB_AFFINITY_HOSTFILE` itself. (An abandoned idea to
convert that file into rankfile form once left two unused forward
declarations here; they have been removed.)

## Things to watch when editing

- **`slot_list` is a fixed `char[64]`** (`RMAPS_RANK_FILE_MAX_SLOTS`).
  Long cpuset expressions can truncate — respect the bound and the
  `snprintf`/copy limits already in the parser.
- **Binding here bypasses `bind_generic`.** rank_file computes
  `proc->cpuset` directly from the slot list and *consumes*
  `node->available`. Don't also route these procs through the
  object-based binder.
- **`+nK` relative indices** are resolved against the *current node
  list*, whose ordering depends on `get_target_nodes`. Changing node
  ordering upstream can silently move `+nK` placements.
- **Generated lexer files:** edit `rmaps_rank_file_lex.l` only; the `.c`
  and `.h` are autogenerated and must not be hand-modified (a project-wide
  rule).
- **Free `rankfile` on every exit** and release every `rankmap` entry;
  both the success and `error:` paths do this.
- Honor `options->app_idx`, the `initial_map = (0 == num_nodes)`
  convention, and the defer-don't-error gate as in every mapper.
