## What this framework does

`prtebacktrace` is a tiny, single-purpose framework: it produces a
**symbolic stack trace of the current thread** on demand. It is the
platform-abstraction layer behind the crash-diagnostics machinery in
[`src/util/stacktrace.c`](../../util/stacktrace.c) — when a PRRTE
process dies on a fatal signal (`SIGSEGV`, `SIGBUS`, `SIGFPE`,
`SIGILL`, …), the installed signal handler dumps a backtrace so a
developer can see where the process was executing.

It exposes exactly **two functions** to the rest of PRRTE:

| Function | Purpose |
|----------|---------|
| `prte_backtrace_print(FILE *file, char *prefix, int strip)` | Write the trace, one frame per line, directly to a file/descriptor. The signal-handler path. |
| `prte_backtrace_buffer(char ***messages, int *len)` | Return the trace as a malloc'd array of strings for the caller to format and free. The non-signal path. |

Both are **plain global symbols**, not a module vtable (see "The
contract" below). The chosen component *is* the definition of these two
symbols in the final binary.

### Where it runs and when it is invoked

- **Any process** links this framework — HNP (`prte`), daemons
  (`prted`), and tools all pull in `src/util/stacktrace.c` and therefore
  the backtrace component.
- The framework is opened once during startup, in
  [`src/runtime/prte_init.c`](../../runtime/prte_init.c)
  (`pmix_mca_base_framework_open(&prte_prtebacktrace_base_framework, …)`).
- The hot invocation is from a **signal handler**:
  `show_stackframe()` in `src/util/stacktrace.c`, registered by
  `prte_util_register_stackhandlers()`, calls
  `prte_backtrace_print(NULL, prefix, 2)` from inside the handler. This
  is the constraint that dominates the whole framework — see
  **Async-signal-safety** below.
- Non-handler callers also exist: `prte_stackframe_output()` /
  `prte_stackframe_output_string()` use `prte_backtrace_buffer()`, and
  [`src/rml/oob/oob_tcp_common.c`](../../rml/oob/oob_tcp_common.c) calls
  `prte_backtrace_print(stderr, NULL, 1)` on an OOB error.

The whole feature is compiled only when `PRTE_WANT_PRETTY_PRINT_STACKTRACE`
is set; the framework itself is always built (the `none` component
guarantees the two symbols resolve on every platform).

## Directory layout

```
prtebacktrace/
  prtebacktrace.h            # the interface: the two fn prototypes + version macro. No module struct.
  configure.m4              # sets STOP_AT_FIRST — exactly ONE component is built
  base/
    base.h                  # declares prte_prtebacktrace_base_framework
    backtrace_component.c   # framework declaration (default register/open/close — all NULL)
    static-components.h     # generated by configure: the single selected component
  execinfo/                 # glibc/BSD backtrace(3) strategy   (priority 30)
  printstack/               # Solaris/illumos printstack(3C)    (priority 30)
  none/                     # no-op stub, returns NOT_IMPLEMENTED (priority 0)
```

Read `prtebacktrace.h` first — it is 75 lines and is the entire
contract. There is no `_types.h`, no scratch struct, and no per-job
state; this framework is stateless.

## The contract — link-time components, not a vtable

Most MCA frameworks define a module struct full of function pointers and
select a winner at runtime. `prtebacktrace` does **not**. As the comment
in `prtebacktrace.h` puts it, "there are two interface functions, both
of which are called directly. The joy of link-time components."

- The component struct is a bare
  `prte_backtrace_base_component_t` — a `typedef` for
  `pmix_mca_base_component_t` — with **no `query`, no module, no
  function pointers.** It carries only the version/name header
  (`PRTE_BACKTRACE_BASE_VERSION_2_0_0`, which expands to
  `PRTE_MCA_BASE_VERSION_3_0_0("backtrace", 2, 0, 0)`).
- Each component provides the *same two global symbols*,
  `prte_backtrace_print` and `prte_backtrace_buffer`. Because these are
  fixed names, **only one component can be linked into a binary** — two
  would collide at link time.
- Therefore selection happens at **build/configure time**, not runtime.
  The framework's `configure.m4` sets `MCA_prte_prtebacktrace_CONFIGURE_MODE`
  to `STOP_AT_FIRST`: configure walks the components in priority order,
  and the first whose availability check passes is the only one built.
  `static-components.h` then lists that single component.

### Return protocol

Both functions return an `int` PRRTE status:

| Return | Meaning |
|--------|---------|
| `PRTE_SUCCESS` | Trace produced (printed, or buffer filled). |
| `PRTE_ERR_NOT_IMPLEMENTED` | This platform's component can't produce a trace (the `none` stub, and `printstack`'s buffer path). |
| `PRTE_ERR_BAD_PARAM` | Bad output descriptor (execinfo, when no usable fd). |

`prte_backtrace_print` parameters:

- `file` — a `FILE *` to write to. If `NULL`, the component falls back to
  the descriptor in the global `prte_stacktrace_output_fileno`
  (declared in [`src/util/stacktrace.h`](../../util/stacktrace.h)),
  which the caller sets up before invoking (stdout/stderr/a file).
- `prefix` — a string emitted before each frame line (PRRTE uses a
  `hostname:pid` banner so multi-process traces interleave legibly). May
  be `NULL`.
- `strip` — number of leading stack frames to omit, so the trace starts
  at the interesting frame and hides the handler plumbing. The signal
  handler passes `2` (skip the handler and this function).

`prte_backtrace_buffer(messages, len)` allocates `*messages` (the caller
frees it) and sets `*len` to the frame count.

## What `base/` provides

The base is deliberately minimal — there are **no base helper
functions** to walk, because there is no runtime module to select or
drive. `base/` contains only the framework object itself:

- `base/backtrace_component.c` declares the framework via
  `PMIX_MCA_BASE_FRAMEWORK_DECLARE(prte, prtebacktrace, NULL, NULL, NULL,
  NULL, prte_prtebacktrace_base_static_components,
  PMIX_MCA_BASE_FRAMEWORK_FLAG_DEFAULT)`. All four hook slots
  (register / open / close / query) are `NULL`: the framework uses the
  MCA defaults, which merely open the static component list and register
  the stock `prtebacktrace_base_verbose` MCA parameter. It does **not**
  run a selection loop or store a "selected module," because the
  interface is resolved at link time (see above).
- `base/base.h` exports the one global,
  `prte_prtebacktrace_base_framework`, which
  `src/runtime/prte_init.c` opens and `src/include/prte_frameworks.c`
  lists among the frameworks.
- `base/static-components.h` is **generated by configure** — do not edit
  it. On a platform with glibc `backtrace()`, for example, it lists only
  `prte_mca_prtebacktrace_execinfo_component`.

That is the whole base. If you came here looking for a
`prte_prtebacktrace_base_select()` or a set of support functions like
the `rmaps` base has, there deliberately isn't one.

## Component selection

Selection is by **configure-time priority**, resolved by `STOP_AT_FIRST`:

| Component | Priority | Availability check (in its `configure.m4`) |
|-----------|----------|--------------------------------------------|
| `execinfo` | 30 | `execinfo.h` present **and** `backtrace` links (libc or `-lexecinfo` on FreeBSD). |
| `printstack` | 30 | `ucontext.h` present **and** `printstack()` links (Solaris/illumos). |
| `none` | 0 | always succeeds — the guaranteed fallback. |

On a given platform normally only one of `execinfo`/`printstack` passes
its check, so the tie at priority 30 is moot; whichever is available
wins, and `none` (priority 0) catches platforms with neither. Every
component builds `static` (each `configure.m4` forces
`COMPILE_MODE = static`) — this framework is not a DSO candidate,
because the symbols must be present at link time.

There is **no runtime MCA parameter to pick a component** — you cannot
switch backtrace strategies at launch, only at build time. The only
runtime knob in this area is the *output target*
(`prte_stacktrace_output_filename` → `stdout`/`stderr`/`file[:name]`/
`none`), which lives in `src/util/stacktrace.c`, not in this framework.

## Conventions and gotchas

- **Async-signal-safety is the prime directive.** The primary caller is
  a signal handler firing on `SIGSEGV`/`SIGBUS`/etc. Inside
  `prte_backtrace_print` you may call **only** async-signal-safe
  functions — no `malloc`, no `printf`/`fprintf`, no locking. `execinfo`
  obeys this by using `backtrace()` + `backtrace_symbols_fd()` and
  writing through `pmix_fd_write()` straight to the descriptor; it
  never allocates. `prte_backtrace_buffer`, by contrast, *does* allocate
  (`backtrace_symbols`), so `prtebacktrace.h` explicitly warns it is
  "probably bad to call from a signal handler" — keep it on the
  non-handler paths only.
- **`strip` counts frames from the top**, i.e. the most recent calls.
  Adding or removing a wrapper frame between the handler and
  `prte_backtrace_print` changes the correct `strip` value; the caller
  in `stacktrace.c` passes `2` on purpose.
- **Honor the `file == NULL` → `prte_stacktrace_output_fileno`
  fallback.** Callers rely on it to redirect crash output to a
  per-rank file. Return `PRTE_ERR_BAD_PARAM` if the resulting descriptor
  is `-1` rather than writing to a bad fd.
- **Every component must define both symbols**, even if one just returns
  `PRTE_ERR_NOT_IMPLEMENTED` (as `printstack`'s buffer path and all of
  `none` do). A missing symbol breaks the link on that platform.
- **`static-components.h` is generated** — never hand-edit it, and never
  edit the per-component `Makefile`/`Makefile.in` (only `Makefile.am`).
- Standard PRRTE rules still apply: `prte_config.h` first, braces on
  every block, `NULL ==` / constant-on-left comparisons, no new compiler
  warnings, `PRTE_` / `prte_` symbol prefixes.

## Debugging

```sh
# Force a per-rank crash-trace file to watch the framework fire:
prun --prtemca stacktrace_output file:/tmp/trace ...

# Framework open/registration chatter (stock base verbosity knob):
prte --prtemca prtebacktrace_base_verbose 5 ...

# Confirm which component was compiled in on this platform:
prte_info | grep prtebacktrace
```

If a crash produces no trace, first check that the build set
`PRTE_WANT_PRETTY_PRINT_STACKTRACE`, then check which component landed in
`static-components.h` — a platform that fell through to `none` yields no
trace by design.
